# FPGA Accelerator Linux-on-SD Boot

## Hardware

- **Board**: MicroPhase Z7-Lite (Zynq 7010, xc7z010clg400-1)
- **DDR3**: 512 MB at 0x00100000–0x20000000
- **PL clock**: 100 MHz (FCLK_CLK0)
- **FPGA IP**: hp_fsm_top at AXI4-Lite 0x43C00000 (GP0)
- **UART**: UART0 at 0xE0000000, MIO 14/15, 115200 baud
- **SD**: SD0 at 0xE0100000, MIO 40-45

## Boot Files (linux/boot/)

- `system_wrapper.bit` — bitstream (generated by Vivado on Windows)
- `matmul_bd.xsa` — hardware handoff (generated by Vivado on Windows)
- `BOOT.BIN` — generated from (FSBL + bitstream + U-Boot), use bootgen on Windows
- `uImage` — Linux kernel (build on Mac)
- `devicetree.dtb` — device tree blob (build on Mac)
- `uramdisk.image.gz` — rootfs (build on Mac)

## Build Instructions (on Mac)

### Prerequisites

```bash
brew install arm-linux-gnueabihf-binutils arm-linux-gnueabihf-gcc
```

### 1. U-Boot

```bash
git clone https://github.com/Xilinx/u-boot-xlnx
cd u-boot-xlnx
export CROSS_COMPILE=arm-linux-gnueabihf-
make zynq_microzed_config   # or zynq_zc702_config
make -j4
# Output: u-boot, spl/u-boot-spl.bin
```

### 2. Linux Kernel

```bash
git clone https://github.com/Xilinx/linux-xlnx
cd linux-xlnx
export CROSS_COMPILE=arm-linux-gnueabihf-
make ARCH=arm xilinx_zynq_defconfig
make ARCH=arm menuconfig  # optional: enable FPGA manager, /dev/mem
make -j4 ARCH=arm UIMAGE_LOADADDR=0x8000 uImage
# Output: arch/arm/boot/uImage
```

### 3. Device Tree

Edit `arch/arm/boot/dts/zynq-microzed.dts` (or our custom .dts) to add:

```dts
/ {
    amba_pl: amba_pl {
        #address-cells = <1>;
        #size-cells = <1>;
        compatible = "simple-bus";
        ranges;
        
        matmul_top: matmul@43c00000 {
            compatible = "xlnx,matmul-top-1.0";
            reg = <0x43c00000 0x10000>;
        };
    };
};
```

Build:
```bash
make ARCH=arm dtbs
# Output: arch/arm/boot/dts/zynq-microzed.dtb
```

### 4. Rootfs with Buildroot

```bash
git clone https://github.com/buildroot/buildroot
cd buildroot
make qemu_arm_vexpress_defconfig
make menuconfig  # set Target options → ARM (little endian) → Cortex-A9/NEON/VFPv3
                 # set Toolchain → Enable VFP extension
                 # set System configuration → hostname, root password
                 # set Target packages → Busybox
make -j4
# Output: output/images/rootfs.cpio.uboot
```

### 5. Test Program

Edit `tmac_linux.c` — port of `tmac_baremetal.cpp` to Linux userspace:
- Replace `reg_read32`/`reg_write32` with `/dev/mem` mmap
- Use `printf` instead of `uart_puts`
- Link with `-lm` for math functions (no custom expf/sinf needed)

```bash
arm-linux-gnueabihf-gcc -O2 -o tmac tmac_linux.c -lm
```

### 6. Create BOOT.BIN (on Windows with Vivado)

```bash
cd D:\Users\u\tmac-zynq-fpga\linux\boot
bootgen -image boot.bif -o BOOT.BIN -w
```

Where `boot.bif` contains:
```
the_ROM_image:
{
    [bootloader] fsbl.elf
    system_wrapper.bit
    u-boot.elf
}
```

### 7. SD Card Layout

```
Partition 1 (FAT32, ~64MB):
  BOOT.BIN
  uImage
  devicetree.dtb
  uramdisk.image.gz

Partition 2 (ext4, rest):
  model.tmac
  tmac
```

## Test Program Porting Notes

The bare-metal `tmac_baremetal.cpp` needs these changes for Linux:

| Bare-metal | Linux equivalent |
|-----------|-----------------|
| `*(volatile uint32_t*)(IP_BASE+off)` | `mmap(0x43C00000, ...)` via `/dev/mem` |
| `uart_puts()` | `printf()` |
| `custom expf()` | `expf()` from `libm` (`#include <math.h>`) |
| `custom sinf()` | `sinf()` from `libm` |
| Pointer-based DDR access | Same (mmap or direct pointer, caches enabled) |
| IRQ handler | `/dev/uio` or polling (polling simpler) |
| `dsb` barriers | `__sync_synchronize()` or `msync()` |

## FPGA Register Access via /dev/mem

```c
#include <fcntl.h>
#include <sys/mman.h>

volatile uint32_t* gp0_map(void) {
    int fd = open("/dev/mem", O_RDWR | O_SYNC);
    if (fd < 0) { perror("/dev/mem"); return NULL; }
    volatile uint32_t* p = mmap(NULL, 0x10000, PROT_READ|PROT_WRITE,
                                 MAP_SHARED, fd, 0x43C00000);
    close(fd);
    if (p == MAP_FAILED) { perror("mmap"); return NULL; }
    return p;
}
```
