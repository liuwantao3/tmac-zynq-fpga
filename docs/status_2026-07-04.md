## Track A: Maximize BRAM Utilization (Free LUTs)

Goal: convert all remaining distributed-RAM arrays in Q8 core to BRAM, freeing ~1,500 LUTs.

### Current BRAM Allocation (9 BRAM18s = 4.5 tiles)
| Memory | Size | BRAM18s | Type |
|--------|------|---------|------|
| wmem | 512×64 | 1 | BRAM ✅ |
| acc_b0..b7 | 8× 512×48 | 8 | BRAM ✅ |
| smem | 128×16, 8-port | 0 (LUTRAM) | ❌ |
| act_reg | 64×16, 1-port | 0 (LUTRAM) | ❌ |

### Proposed (25 BRAM18s = 12.5 tiles, 20.8%)

#### 1. smem → 8× BRAM banks (8 BRAM18s, saves ~550 LUTs)
- Split by column lane: bank wi_i = scales for lane wi_i
- Read address: `{g, k[5]}` (4 bits, shared across banks)
- Write: `sc_addr[3:1]` selects bank, `{sc_addr[6:4], sc_addr[0]}` = local addr
- Each bank = 16×16-bit (padded to 512×18 BRAM18)
- Pipeline: add PRE stage to register addresses 1 cycle ahead

#### 2. act_reg → BRAM (1 BRAM18, saves ~32 LUTs)
- 64×16 = 1024 bits → 512×18 BRAM18
- Single read port, single write port
- Pipeline: read address registered in PRE stage, output captured 1 cycle later

#### 3. wmem → 8× BRAM banks (7 new BRAM18s, saves ~200 LUTs)
- Currently 1 BRAM (512×64). Output byte-sliced: `wmem_rdata[wi_i*8 +: 8]`.
- **Proposed**: 8 banks of 512×8 (padded to 512×18), one per column lane.
- Each bank has 1 read port → no byte-slicing needed, direct byte output.
- Write: broadcast 64-bit word to all 8 banks (`bank_i <= wt_din[i*8+7:i*8]`).
- Simplifies read path: each lane reads directly from its bank output.
- **Note**: wmem is already BRAM — this saves byte-slice mux logic, not LUTRAMs.

#### 4. Pipeline Adaptation
- Add PRE stage (register read addresses for all BRAMs)
- Extend from 5-stage to 6-stage pipeline
- DRAIN states: DRAIN→DRAIN2→DRAIN3→DRAIN4 (was DRAIN3)
- Total compute: 64×8 + 6 = 518 cycles (was 516)

### Estimated Savings

## Track B: Wider MAC (DSP Parallelism)

Goal: increase column-parallel MAC from 8 cols/cycle to 16 or 32 cols/cycle, using more DSPs.

### Current
- 8 column lanes → 8 MACs/cycle
- 16 DSPs: 8 for dequant (24-bit signed multiply) + 8 for product multiply (16×16)
- 64 columns per group → 64×8 = 512 cycles per 64×64 tile

### Options

#### Option B1: 16× column (2× throughput, +16 DSPs)
- 16 parallel MAC units → 32 DSPs total (40%)
- 64 columns → 32 iterations per group
- 64×32 + drain = ~260 cycles per 64×64 tile (2× speedup)
- Add ~3,000 LUTs for second 8-lane datapath

#### Option B2: 32× column (4× throughput, +48 DSPs)
- 32 parallel MAC units → 64 DSPs total (80%)
- 64 columns → 16 iterations per group
- 64×16 + drain = ~135 cycles per 64×64 tile (4× speedup)
- Add ~6,000 LUTs

### Key Bottleneck
The Q8 core already uses 50% of design LUTs (~5,651). After Track A frees ~1,500 LUTs, Track B's added LUTs would bring utilization to:
- Option B1: ~10,500 + 3,000 = 13,500 LUTs (77%) — feasible
- Option B2: ~10,500 + 6,000 = 16,500 LUTs (94%) — tight
